# 学习笔记 — Live2D Viewer 项目

## 项目概览

一个在浏览器里查看 Live2D 模型的网页应用。用 Vite 做开发服务器、PixiJS 做 2D 渲染、pixi-live2d-display 做 Live2D 桥接。鼠标可以拖拽/点击模型、键盘切动作表情、滚轮缩放、拖 zip 替换模型。

---

## 1. 技术栈是怎么串起来的

### 1.1 渲染层：PixiJS 6

PixiJS 是一个 WebGL 2D 渲染库。它把 WebGL 复杂的 GPU 操作封装成简单的"舞台 → 精灵/模型"层级结构，你只管往舞台上加东西，不用写着色器。

这个项目没直接用 `<canvas>` 手绘，因为 Live2D 模型本质是一堆带纹理的变形网格，自己写 WebGL 渲染太复杂了。PixiJS 已经做了：

- **Application**：画布容器，管理 renderer + stage + ticker
- **stage**：根显示对象，所有可见元素都 addChild 到它上面
- **ticker**：帧循环，每秒 60 帧，每帧更新 stage 上所有对象

```typescript
const app = new Application({
  width: 1920,
  height: 1080,
  backgroundColor: 0x1a1a2e,
  resolution: window.devicePixelRatio || 2,  // 高分屏适配
  autoDensity: true,  // CSS 尺寸 = 逻辑尺寸，像素密度由 resolution 处理
})
app.stage.addChild(model)  // 把 Live2D 模型放到舞台上
```

### 1.2 Live2D 桥接层：pixi-live2d-display

这个包做了两件事：

1. **把 Live2D 模型变成 PixiJS 显示对象**（继承 PIXI.DisplayObject），所以能用 `app.stage.addChild()`
2. **封装了 Live2D Cubism Core 的底层 API**，对外暴露出简单的 `motion()`、`expression()`、`focus()`、`tap()` 方法

它依赖一个全局变量：`Live2DCubismCore`，这个变量由 CDN 脚本提供。

### 1.3 Live2D 核心层：Cubism Core

这是 Live2D 公司专有的计算引擎（WebAssembly + JS 封装），负责实际的数学模型计算：

- **网格变形**：根据参数值对模型顶点做插值变换（参数是 0-1 的浮点数，比如 `ParamAngleX` 控制头左右转）
- **物理模拟**：头发、裙摆的摆动效果
- **姿态叠加**：修正叠加姿态时的角度偏差

它通过 `<script>` 标签在 HTML 里最先加载，因为后面的代码都依赖它。

### 1.4 加载链路

```
npm run dev (Vite)
  → 浏览器打开 index.html
    → <script> 加载 live2dcubismcore.min.js（CDN，Live2D 官方核心）
    → <script type="module"> 加载 src/main.ts
      → main.ts 顶层执行 setupZipLoader()（注入 JSZip）
      → main() 异步执行
        → fetch('/src/models/waifu-flat.zip')（Vite 静态资源服务）
        → loader.ts 解压 zip → 构造 Cubism4ModelSettings
        → pixi-live2d-display 用这些配置初始化 Live2D 模型
        → 挂到 PixiJS stage 上
        → 每帧 ticker 驱动模型更新 → WebGL 绘制到 canvas
```

## 2. 交互系统是怎么设计的

### 2.1 为什么不用自带的交互

pixi-live2d-display 自带一套交互系统（`autoInteract: true`），包括自动眨眼、头跟踪、点击动作等。但这个项目把它关了：

```typescript
await Live2DFactory.setupLive2DModel(model, [zipFile], { autoInteract: false })
```

原因是**需要更精确的控制**：拖拽移动模型是自带交互做不到的，而且在 stage 层面统一处理 pointer 比让模型内部自己处理更灵活。

### 2.2 自定义 pointer 处理（pointer.ts）

核心是一个简单的**状态机**：

```
          pointerdown
  空闲 ─────────────→ 按下（记录起点）
                         │
                   pointermove
                   距离 > 5px ?
                   ├─ 否 → 只跟踪视线
                   └─ 是 → 拖拽（移动模型位置）
                         │
                   pointerup
  拖拽结束 ←─────────┤
  触发 tap ←─────────┘
```

**5px 阈值**很重要：手指/鼠标按下时总会有微小抖动，不加阈值 tap 几乎无法触发。

**视线追踪**：无论是否在拖拽，`pointermove` 每帧都调 `model.focus(x, y, true)`。第三个参数 `true` 是翻转 Y 轴——PixiJS 的 Y 向下，但 Live2D 期望屏幕坐标系。

**pointerupoutside**：如果鼠标按在模型上，拖到浏览器外面松手，`pointerup` 事件不会触发，但 `pointerupoutside` 会。没有这个监听，`pointerDown` 永远卡在 `true`，下次鼠标移回来直接就"拖拽"了。

### 2.3 键盘映射（main.ts）

```typescript
1-9     → 前 9 个 motion（跳过 idle 组，因为那个是自动循环的）
Q-P     → 前 10 个 expression（键位顺序 qwertyuiop）
0       → 停止所有动作 + 恢复默认表情
```

**焦点检测**：如果用户正在输入框/文本域/下拉框里打字，按键不触发模型动作。这是通过检查 `e.target` 的标签名实现的。

**防止重复绑定**：`currentKeyHandler` 是一个模块级变量，每次 `setupKeyboard` 时先移除旧的 `keydown` 监听器，再绑定新的。否则每换一次模型就多绑一层，按一次键触发多次。

## 3. 模型加载的核心难点

### 3.1 ZipLoader 注入

pixi-live2d-display 默认用 `fetch` 逐个读取 zip 里的文件。这个项目用 JSZip 替代：

```typescript
ZipLoader.zipReader = (data, _url) => JSZip.loadAsync(data)
```

同时还覆盖了 `readText`、`getFilePaths`、`getFiles` 三个静态方法。这是一次性操作，用 `zipLoaderReady` 标记防止重复执行。

### 3.2 松散文件补全（最关键的创新）

**问题**：很多社区制作的 Live2D 模型的 `.model3.json`（配置文件）没有列出所有的 `.motion3.json`（动作文件）和 `.exp3.json`（表情文件）。结果是这些动作/表情文件存在但不能用。

**解决思路**：
1. 先走默认流程拿到 settings
2. 遍历 zip 里所有文件，找出 `.mtn` / `.motion3.json` / `.exp3.json`
3. 检查哪些文件没在 settings 里被引用
4. 追加到 settings 的 motions 和 expressions 列表
5. **重建 idle 组**：pixi-live2d-display 默认循环播放 idle 组的 motion，如果不重建，补全的动作就不会自动播放

**没有 `.model3.json` 的情况**：有些模型的 zip 里直接就是 `.moc3` + `.png` + 其他文件，没有 json 配置。这时候就根据文件扩展名自动推断：
- `.moc3` → Moc（模型数据文件）
- `.png` → Textures（贴图）
- 文件名含 `physics` → Physics（物理配置）
- 文件名含 `pose` → Pose（姿态配置）

### 3.3 URI 编码 middleware

pixi-live2d-display 用 middleware 链处理模型加载流程。如果 zip 里的文件名含有中文或特殊字符，fetch 请求会 404。

解决方案：在 FileLoader middleware **之前**插入一个编码 middleware：

```typescript
// 对 moc、textures、url 做 encodeURI
s.moc = encodeURI(s.moc)
s.textures[i] = encodeURI(s.textures[i])
```

插入位置很重要——必须在 FileLoader 前面，因为 FileLoader 会直接发 fetch。

### 3.4 模型替换时的资源清理

```typescript
export async function replaceModel(app, zipFile) {
  for (let i = app.stage.children.length - 1; i >= 0; i--) {
    const child = app.stage.children[i]
    if (child instanceof Live2DModel) {
      child.destroy()           // 释放 WebGL 纹理、motion 计时器等内部资源
      app.stage.removeChild(child)  // 从显示树移除
    }
  }
  return loadModelFromFile(app, zipFile)
}
```

`destroy()` 不能省，否则切换多次模型后 GPU 纹理占满显存，页面会崩溃。

## 4. 控制面板的设计模式

### 4.1 纯 DOM，零框架

刻意不用 Vue/React，直接操作原生 DOM。对于一个有 4 个滑块 + 2 个按钮 + 动态列表的面板来说，Vanilla JS 完全够用，多引入一个框架反而不值。

### 4.2 回调模式

```typescript
interface PanelCallbacks {
  onPositionChange: (x: number, y: number) => void
  onScaleChange: (scale: number) => void
  onResetPosition: () => void
  onFitToScreen: () => void
  onImportFile: (file: File) => void
}
```

面板只负责 UI 渲染和用户操作捕获，业务逻辑通过回调交给 `main.ts` 处理。这是典型的**控制反转**——UI 组件不依赖业务逻辑，可以被测试和复用。

### 4.3 单向数据流

```
main.ts（状态持有者）── 同步 ──→ ControlPanel（纯展示）
        ←── 回调 ──
```

模型的位置、缩放、名称变更后，main.ts 主动调用 `panel.setPosition()`、`panel.setScale()` 等方法同步 UI。面板不主动读模型状态。这个单向流动让状态管理更容易 debug——状态只有一处源头。

### 4.4 innerHTML 一次性构建

```typescript
el.innerHTML = `<h3>...</h3><div class="section">...</div>...`
this._xSlider = el.querySelector('[data-axis="x"]')!
```

整个面板的 DOM 结构用一次 `innerHTML` 赋值全部创建，然后用 `querySelector` 缓存关键引用。这比逐个 `createElement` + `appendChild` 更快更可读，代价是有潜在的 XSS 风险（但这里没用户输入的内容插入 HTML，所以安全）。

## 5. 开发工具链

### 5.1 Vite

| 配置 | 含义 |
|------|------|
| `assetsInclude: ['**/*.zip']` | 把 .zip 当静态资源，能被 fetch 加载 |
| `server.host: true` | 局域网可访问 |
| `server.watch.ignored` | 排除无关目录的监视 |
| `optimizeDeps.entries` | 预构建 index.html 的依赖 |
| `build.assetsInlineLimit: 0` | 不内联任何资源文件 |

关键理解：Vite 开发时不打包，用浏览器原生 ES Module 按需加载。改了代码 Vite 只重新编译被改的那一个文件，其余不动，所以热更新极快。

### 5.2 TypeScript

tsconfig 的核心设置 `noEmit: true` 意味着 TypeScript **只管类型检查**，不生成 JS 文件。实际编译由 Vite 的 esbuild 完成。这是现代前端项目的标准做法：类型检查用 tsc（准确但慢），打包用 esbuild（快但不做类型检查）。

## 6. 从代码中学到的技巧

### 6.1 模块级 guards

```typescript
let zipLoaderReady = false
export function setupZipLoader() {
  if (zipLoaderReady) return
  zipLoaderReady = true
  // ...
}
```

不需要单例模式——模块天然就是单例（ES Module 只执行一次）。用布尔标记就够了。

### 6.2 事件监听器生命周期管理

```typescript
let currentKeyHandler = null
function setupKeyboard(model, infoEl) {
  if (currentKeyHandler) {
    window.removeEventListener('keydown', currentKeyHandler)
  }
  const handler = (e) => { /* ... */ }
  currentKeyHandler = handler
  window.addEventListener('keydown', handler)
}
```

在全局对象上绑事件，别忘了换模型时把旧的解绑。把引用存在模块变量里，下次就能解绑。

### 6.3 文件输入的隐藏技巧

```html
<input type="file" accept=".zip" style="display:none" id="file-input">
<span class="btn-file">📦 选择 .zip 文件</span>
```

```typescript
btn.addEventListener('click', () => fileInput.click())
fileInput.addEventListener('change', () => {
  const file = fileInput.files?.[0]
  if (file) { /* 处理文件 */ }
  fileInput.value = '' // 重置，这样选同一个文件也能再次触发 change
})
```

隐藏难看的原生文件选择框，用自定义按钮触发。选完后清空 value 值，否则选同一个文件不会触发 change 事件。

### 6.4 状态同步的辅助函数

```typescript
function syncPanelMotions(model, panel) {
  panel.showMotionExpressionInfo(getMotionEntries(model), getExpressionNames(model))
}
```

模型替换后，新模型的 motion/expression 列表不同了。这个函数负责把最新的列表同步到面板显示。在文件导入和拖放两处都调用它，避免了直接写重复代码。

## 7. 已发现的改进方向

1. **滚轮步进硬编码**：`main.ts:103` 的 `0.05` 应引用 `config.ts` 的 `SCALE_STEP`
2. **窗口缩小时未适配**：resize 只调了渲染器尺寸，没调 `fitToScreen`
3. **类型安全**：多处用 `(model as any).internalModel` 绕过类型检查，可扩展类型声明
4. **重复逻辑**：模型替换后的面板同步逻辑在文件导入和拖放两处重复

## 8. 关键概念速查

| 概念 | 一句话解释 |
|------|-----------|
| PixiJS Application | 画布 + 渲染器 + 帧循环的封装 |
| PixiJS stage | 根显示对象，所有可见元素挂它下面 |
| PixiJS ticker | 帧循环驱动器，每帧更新所有对象 |
| Live2D Cubism Core | 官方底层引擎，WebAssembly + JS，处理网格变形/物理/姿态 |
| pixi-live2d-display | 社区桥接层，把 Live2D 模型变成 PixiJS 显示对象 |
| Cubism4ModelSettings | 模型配置文件对象，定义 moc/贴图/动作/表情的路径 |
| ZipLoader | pixi-live2d-display 的 zip 加载器，用 fetch 读文件 |
| JSZip | 纯 JS 的 zip 库，可以解析 zip 文件结构 |
| Middleware | 模型加载流程中的拦截器链，可在加载前后插入逻辑 |
| idle group | pixi-live2d-display 默认循环播放的 motion 组 |
| `model.focus(x, y, invertY)` | 让模型视线追踪某个屏幕坐标 |
| `model.tap(x, y)` | 模拟点击模型，触发 hit 检测 |
| `model.motion(group, index)` | 播放某个动作组中的指定动作 |
| `model.expression(name)` | 切换到指定表情 |
